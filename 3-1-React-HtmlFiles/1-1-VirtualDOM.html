Virtual DOM vs Real DOM

  1. DOM - Document Object Model.

  2. The virtual DOM is a programming concept.
  3. and The virtual DOM is an virtual representation of a UI is kept in memory.
  4. and it will allways synced with the real DOM by using ReactDOM library.
  5. This process is called reconciliation.

  6. Every react component hsa state and render action.
  7. This state here is the data that we want to display when the component is rendered.
  8. And the render method, as you can tell, is responsible for describing what the Ul should look like.
  9. The output of this render method is a React element, which is a simple plain JavaScript object that maps to a DOM element.
  10. It's not a real DOM element.
  11. It's just a plain JavaScript object that represents that DOM element in memory.
  12. So react keeps a lightweight representation of the DOM in memory, which we'll refer to as the virtual DOM.
  13. Unlike the browser or the real DOM, this virtual DOM is cheap to create.
  14. when we change the state of a component, we get a new React element.
  15. React will then compare this element and its children with the previous one.
  16. It figures out what has changed, and then it will update a part of the real DOM to keep it in sync with the virtual DOM.
